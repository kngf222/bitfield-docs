# Placeable surfaces.

<p className="bf-lead">
A placeable surface is a product UI contract. It says what can appear, where it can appear, and which prepared reads or targets the surface may use. The shell reads that contract and places the surface without learning the product story behind it.
</p>

## What this explains

This page explains the primitive. It is not the cookbook.

Use this page when you need to understand why the architecture stays clean. Use [Placeable surface product loop](/cookbook/placeable-surface-product-loop) when you want the full example succession with files and code.

## The separation

The shell should not know what a launch dashboard is. It should not know what support means. It should not know which package owns help search.

The shell should only know this:

| Generic thing | What it means |
|---|---|
| Surface descriptor | A piece of UI that can be placed in a region. |
| Region | A named area such as `main`, `rail`, or `panel`. |
| Prepared input names | Data names the surface is allowed to read. |
| Target names | Work names the surface is allowed to request. |
| Component key | The renderable body for that surface. |

That is the same class of idea as a command center: the arrangement model is generic, the sidebar is generic, and the product meaning lives behind descriptors.

## The product map

```text
Package set
  -> packages
  -> prepared inputs
  -> targets
  -> surface descriptors

Product shell
  -> reads surface descriptors
  -> renders a generic sidebar from labels and order
  -> renders the active surface in a generic region

Surface body
  -> reads only the prepared inputs named by its descriptor
  -> calls only the targets named by its descriptor
```

The product becomes larger without making the shell import package setup code or product-specific screen names.

## What this prevents

| Bad shape | Placeable-surface shape |
|---|---|
| Sidebar branches on `dashboard`, `support`, and `help`. | Sidebar renders descriptor labels and active ids. |
| Layout imports product screens directly. | Layout receives descriptors and resolves component keys. |
| One new product area means editing the shell. | One new product area means adding a descriptor and body. |
| AI agents add another special case. | AI agents repeat the same descriptor contract. |

The primitive is boring on purpose. The shell can stay dumb while the product becomes more capable.
