Cookbook entries show a whole shape, not a loose snippet. Each one starts with the file or app surface you own, then shows the Bitfield boundary it talks to.

Use these when you want to copy the shape and adapt the names. The examples stay on the public side of the wall: package files, package-owned content, callable slots, and the two app-facing Runtime Kit calls.

The full examples live in `examples/` and are indexed by `cookbook-examples.json`. The docs check makes sure the snippets shown here match those files exactly.

## Start here

- [Package set with one record](/cookbook/package-set-with-one-record) if you want the smallest complete package.
- [Package-owned file](/cookbook/package-owned-file) if your package needs to bring bytes with it.
- [Callable package slot](/cookbook/callable-package-slot) if your package needs a target that can answer requests.
- [React surface for package data](/cookbook/react-surface-for-package-data) if your app needs to show data and trigger work.
- [Placeable surface product loop](/cookbook/placeable-surface-product-loop) if you want the package-to-shell composition.

## What is not here yet

Account login, credential recovery, key rotation, and account portal flows are not in this cookbook lane yet. Those docs need the final public account contract before they can be written cleanly.
