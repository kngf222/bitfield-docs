The customer needs four answers fast: what costs $0 today, when the plan starts billing, what consumes the included runtime/device range, and where plan management happens.

## Choices at checkout

| Choice | Included runtime/device units | What it means |
|---|---:|---|
| Try Personal free | 1-5 | Start Personal with 3 days free, then billing begins unless you cancel first. |
| Personal | 1-5 | Pay now for the Personal range. |
| Builder | 6-100 | Pay now for the Builder range. |
| 101+ | 101 or more | Use the curve for larger triggered runtime/device counts. |

Monthly and yearly cadence are separate choices. The checkout total should change when the cadence changes.

## What "free trial" means

The free trial is for Personal. It gives you the same runtime/device range as Personal for the trial window. If you keep it after the trial, billing starts on the displayed start date and cadence.

The checkout should say the important part clearly:

<div className="bf-callout">
$0 today, then the Personal price starts after the 3-day trial unless you cancel first.
</div>

## What consumes the included range

The public phrase is active device. The precise meaning is a triggered Bitfield runtime identity in the billing window.

That runtime identity can be a laptop, phone, server, cloud runtime, local box, kiosk, test machine, or another environment that actually triggers Bitfield runtime access.

It is not:

- a dormant install
- a repo with package files
- a website visitor reading static output
- a page view
- a click
- a runtime second

## Larger counts use ranges

The curve is cumulative. That means each range uses its own rate, then the ranges add up.

For example, when a count goes above 100, the first 100 are still the base Builder range. Extra triggered runtime/device units are priced by the next range, and later ranges only apply to the units inside those later ranges.

## Legal pages

Checkout must link the public legal pages:

- [Terms of Service](https://bitfield.so/terms)
- [Refund Policy](https://bitfield.so/refund-policy)
- [Privacy Policy](https://bitfield.so/privacy)

Those pages govern the actual legal terms. This docs page explains the product language so the checkout is easier to understand.

## Common failures

| Symptom | Cause | Fix |
|---|---|---|
| Customer thinks pricing is per person | Active device was read as seat | Say triggered runtime/device unit |
| Customer thinks an old install is charged forever | Triggering and installing were mixed up | Say dormant installs do not count by themselves |
| Customer thinks every visitor is charged | Traffic got mixed up with runtime triggers | Explain that visitors and page views are not triggers |
| Customer accidentally uses Runtime Kit for a static page | Public content and live runtime were not separated | Use static output when the page does not need Bitfield runtime behavior |
| Customer misses the post-trial billing date | Trial language is hidden | Show `$0 today` and the paid start clearly |
| Larger-device pricing feels like a black box | Ranges are not explained as cumulative | Explain that each range applies only to units inside that range |
