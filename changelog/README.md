# Changelog

This folder owns the public Bitfield changelog.

The rendered page is customer-facing. It should only describe changes a reader can act on:

- Start flow changes
- Runtime Kit public docs changes
- Build guide changes
- Account, trial, billing, active-device, and cancellation docs changes
- Proof and benchmark explanation changes
- Docs navigation or page-organization changes

Do not list private engineering chores, repo housekeeping, validation commands, generated page mechanics, unpublished implementation work, or local process details here. Those belong in the project plan and tracking log, not in public docs.

Every entry should answer:

- What changed?
- Why does it matter to the reader?
- Which category does it belong to?
- Which public pages changed?
