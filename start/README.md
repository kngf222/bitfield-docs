# Start Docs

This folder owns the first public onboarding path for Bitfield.

## Page Contract

Start pages must help a new reader move from curiosity to a first visible result. They should not behave like reference pages or marketing pages.

Every Start page should include:

- A concrete reader moment.
- A step-by-step path.
- A visual flow using shared docs classes.
- The expected result.
- First failure modes and fixes.
- Links into Runtime Kit, Activation, Build, Concepts, and Proof when the reader needs depth.

## Pages

| Page | Purpose |
|---|---|
| `quickstart.mdx` | First-result tutorial: key, Runtime Kit, read, request, visible success. |
| `get-your-key.mdx` | Account/key/device path: account portal, email, trial/plan, key access, first activation, recovery. |
