# Concept Docs

This folder owns the public Bitfield concept curriculum.

## Page Contract

Concept pages explain why the product works the way it does. They are not reference pages and not setup guides.

Every concept page should include:

- A concrete product scene.
- A visual flow or structured map.
- A simple model first.
- A technical boundary after the model.
- Common mistakes or wrong turns.
- A next link into the following concept, Runtime Kit, Build, Activation, Proof, or Reference.
