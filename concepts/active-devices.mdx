## What counts

A laptop can count. A phone can count. A tablet can count. A server can count. A Raspberry Pi can count. A runtime environment can count when it uses your key to run Bitfield.

The working rule: if you turn Bitfield on there with your key, and you have not revoked or replaced it, it is active.

## How moving a spot works

If you stop using one device, revoke it in the account portal. That frees the spot so a replacement device can use your key.

## Why pricing uses this word

Bitfield pricing is based on active devices because the product is not priced per app, per project, per seat, token, install, revenue share, or feature. The unit is the device or runtime identity currently allowed to run with your key.

## Common confusion

| Confusion | Correct model |
|---|---|
| "Is an active device the same as a user?" | No. It is the device or runtime identity currently authorized to run with your key. |
| "Does a project count?" | The public pricing word is active device, not project. |
| "Can I move a spot?" | Yes. Revoke or replace the old device in the account portal when that path is available. |
| "Does a server count?" | It can when it runs Bitfield with your key. |
| "Where do I manage this?" | Use [account.bitfield.so](https://account.bitfield.so). |
